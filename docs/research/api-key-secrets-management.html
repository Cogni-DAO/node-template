api keys + recoverable secrets · architecture

version v1 → Walk · proj.agentic-fork-bootstrap
principle No human types a secret VALUE into a UI. App-wide AEAD keys are a single-key blast radius.

decision matrix

agent api keys

today aead-encrypted via CONNECTIONS_ENCRYPTION_KEY
walk sha-256 hash-only · no key
why Stripe / GitHub pattern. Compromise db = irreversible hashes. Removes the single-point-of-failure key.

oauth refresh tokens

today aead-encrypted via CONNECTIONS_ENCRYPTION_KEY
walk vault transit · per-tenant key
why Recoverable → needs reversible crypto. Per-tenant Transit key scopes blast radius. Rotation = transit/rotate; ciphertext stays valid via key-version.

poly clob creds

today aead-encrypted via POLY_WALLET_AEAD_KEY_HEX
walk vault transit · per-tenant key
why Same shape as oauth. Transit fits the hot path; per-tenant key isolates wallets.

flow

anti-patterns

app-wide aead key one key for all tenants → compromise = total population breach. Per-tenant Transit keys replace it.
reversibly-encrypted api keys if recovery isn't needed, don't make it recoverable. Hash-and-compare; never re-derive.
openbao kv read on hot path KV = bootstrap-window writes + ESO sync. Hot-path belongs to Transit or to in-memory state hydrated at pod start.
