## Binding Constraint

> **P01 — Single-operator bottleneck** (5/5) — Last evaluated: 2026-03-25

The meta-constraint. It caps throughput on everything and directly contradicts the DAO mission. Every charter independently reports "100% dependent on one human (Derek)." Zero movement on community targets, channel work, or observability since first evaluation. March 2026 commits show operator time absorbed by infra polish (GitOps Argo CD stack, secret rotation CLI) rather than community-facing work — the pattern that P06 (infrastructure complexity tax) now names explicitly.

The relief path is: M02 (connect channels) → M01 (reach first community members) → P01 (distribute operator load). Vitalik's Feb 2026 proposal for AI stewards validates the project's direction: AI agents can cover routine governance, but only once humans are in the loop to provide the community substrate. External DAO research confirms the critical importance of first 100 members setting culture.

Note: Previous evaluations conflated "First evaluation" with a same-day second run that added no new data. The history below has been corrected.

**Work on now:**

1. `proj.messenger-channels` — Crawl P0 requires only config changes to OpenClaw gateway (add `channels:` section, remove tool deny-list entries, add STATE_DIR volume). No new code infrastructure. Telegram bot token is the fastest path. This is the most leveraged unblocked action available.
2. `proj.context-optimization` — Phase 1 tasks (task.0068-0070) have no owner and no commits. Lower `softTrimRatio` to 0.15 is a 15-minute config change that could cut GOVERN loop cost by 30-50%, making the system more usable while P01 persists.
