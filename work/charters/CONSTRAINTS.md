## Binding Constraint

> **L02 — Scope creep / lack of focus** (5/5) — Last evaluated: 2026-04-23

**Previous evaluations were correct at the time and are now stale.** P01 "single-operator" is no longer the binding constraint: 291 commits in the last month (84 feat + 99 fix) prove the operator has high throughput. The binding constraint is what that throughput is pointed at.

The project is now executing **two incompatible strategies in parallel**:

1. **Stated mission** (ROADMAP.md, all four charters): Node = sovereign DAO+app org-factory. Phase 0 goal = Node Formation MVP. Focus projects = messenger-channels, observability, governance-agents.
2. **Executed mission** (git log since 2026-03-25): `proj.poly-copy-trading` — autonomous Polymarket trading with per-tenant wallets, CLOB redeem, Privy custody, 30s Data-API polling. Created 2026-04-19. Not on the roadmap. Consuming ~70% of recent feature work.

Phase 0 is still unchecked. `proj.messenger-channels` has zero commits. `proj.observability-hardening` is still Paused. Meanwhile, ~40% of commits are CI/CD firefighting (bug.0295/0335/0339/0343/0344/0361) servicing the trading-bot deploy path — P06 has elevated to 5 as the direct cost of running a multi-app Argo CD + Image Updater stack as a solo operator, with Image Updater itself [explicitly flagged as Technology Preview, not production-recommended](https://docs.redhat.com/en/documentation/red_hat_openshift_gitops/1.19/html/release_notes/gitops-release-notes).

External reality check on the Poly path: [only 7.6% of Polymarket wallets are profitable](https://medium.com/technology-hits/why-92-of-polymarket-traders-lose-money-and-how-bots-changed-the-game-2a60cd27df36); [arbitrage windows have collapsed from 12.3s to 2.7s and 73% of profit is captured by sub-100ms bots](https://medium.com/illumination/beyond-simple-arbitrage-4-polymarket-strategies-bots-actually-profit-from-in-2026-ddacc92c5b4f). A 30s-poll retail copy-trader entering this market is structurally disadvantaged — the EV case needs to be explicit before more build cost is spent.

TOC says: you cannot subordinate to a constraint if there is no single goal to subordinate to. Until Derek picks one primary mission, _every_ lower-ranked constraint will continue to score poorly because operator attention is split.

**Work on now — decision before build:**

1. **Reconcile ROADMAP.md with actual execution.** Either: (a) kill or down-prioritize `proj.poly-copy-trading` and return to Phase 0 Node Formation MVP, OR (b) formally promote Poly to the primary product, rewrite ROADMAP phases, update CHARTER mission accordingly, and deprecate the DAO-factory framing until revenue exists. Pick one. Do not continue both.
2. **Stop the CI/CD bleed (P06).** Before any new Poly features, finish closing bug.0361 and consider whether Argo CD Image Updater (Technology Preview) is worth its maintenance cost vs. static digest pins for a single-operator project.
3. **If path (b) is chosen**: run an EV analysis on Poly copy-trading vs. the retail Polymarket numbers above _before_ shipping more phases. A 30s poll against sub-100ms bots is an edge-negative starting position; the charter must answer "why will we be in the 7.6%?"

Only after (1) resolves does it make sense to re-rank M01/M02/L03 — their relief path depends on which mission is canonical.
