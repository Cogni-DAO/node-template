# Handoff: Epoch Approver UI — EIP-712 Signing + Review/Edit/Finalize Admin Panel

## Context

- Task builds `/gov/review` admin page for approvers to review allocations, sign with EIP-712, and finalize epochs
- Backend steps 1-3 (EIP-712 builder, verifyTypedData migration, sign-data endpoint) were done by previous dev
- Steps 4-6 (useSignEpoch hook, /gov/review page, nav tab) are implemented but **BLOCKED by bug.0121**
- **bug.0121**: The allocation edit API only supports per-user granularity — the signed statement is per-claimant. Approvers need row-level line item editing before signing a ledger
- The finalize workflow (Sign & Finalize button) reports success but **the epoch does not actually transition to finalized** — likely Temporal worker not running or misconfigured in dev stack

## Decisions Made

- **EIP-712 over EIP-191** — structured wallet popup + Safe multi-sig compatibility
- **Dedicated `/gov/review` page** — separate from read-only `/gov/epoch`
- **Server component approver gate** — page.tsx checks `getLedgerApprovers()` + session wallet
- **Editing must be at line-item level** — per-user `finalUnits` override is wrong granularity for a ledger review (see bug.0121 design questions)
- **EpochDetail should NOT have editable columns bolted on** — the correct approach requires bug.0121 resolution first (new API + data model for claimant-level overrides)
