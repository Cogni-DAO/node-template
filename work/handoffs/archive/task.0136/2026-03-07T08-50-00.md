# Handoff: Composable DataSource Port + GitHub Webhook Ingestion

## Context

- The attribution ledger needs to ingest GitHub activity (PRs, reviews, issues, comments, pushes) for contributor credit
- Previously poll-only (Temporal schedule → GitHub GraphQL). This branch adds a webhook fast-path alongside poll
- Preview environment was receiving **zero activity events** — root causes: missing env vars, wrong var names in `.env.test`, and no webhook route
- The webhook normalizer was filtering too aggressively (only merged PRs + closed issues). Widened to capture all event types — downstream selection decides what's attributable
- The branch has 16 prior commits implementing the design + implementation + review feedback. This session added normalizer widening, env fixes, and an e2e test (incomplete)

## Current State

- **Done:** Webhook route (`POST /api/internal/webhooks/github`), `GitHubWebhookNormalizer` (PR all actions, reviews, issues all actions, comments, pushes), container wiring, unit tests (21 passing), type-checks clean
- **Done:** `.env.test` var name fix (`GITHUB_*` → `GH_*`), `GH_WEBHOOK_SECRET` added to all env files and examples
- **Not done:** Round-trip e2e test (`tests/external/ingestion/webhook-roundtrip.external.test.ts`) is half-written — DB query method needs fixing (was using `psql` shell-out, switched to drizzle mid-edit, file left in broken state)
- **Not done:** `GH_WEBHOOK_SECRET` value not yet set in `.env.local` (placeholder empty string). Developer needs to get the actual secret from the GitHub App settings
- **Not done:** Pre-existing external test failures in `github-adapter.external.test.ts` and `ledger-collection.external.test.ts` (GitHub App not installed on `Cogni-DAO/test-repo`, missing `getActivityForWindow` method). These predate this branch
- **Not verified:** Full pipeline not tested end-to-end (app not running during this session)

## Decisions Made

- Webhook route lives in Next.js app, not scheduler-worker — see [webhook architecture research](../../docs/research/webhook-ingestion-architecture.md)
- Ingest-all-filter-later: normalizer captures every event type it can parse; epoch evaluation decides what's attributable
- Bot authors filtered at ingestion (data quality — bots lack stable numeric IDs)
- Merged PR event IDs match poll adapter IDs (`github:pr:owner/repo:42`) for natural dedup via `ON CONFLICT DO NOTHING`
- Non-terminal PR/issue actions get action-suffixed IDs (`github:pr:owner/repo:42:opened`) to avoid colliding with the canonical merged/closed ID
