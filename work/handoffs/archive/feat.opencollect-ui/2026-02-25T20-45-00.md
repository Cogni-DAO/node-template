# Handoff: Sidebar Navigation + Layout Regression Fixes

## Context

- Adopting a left-side vertical sidebar (like OpenCollective/ChatGPT) for authenticated `(app)` pages
- Public pages keep `AppHeader` + `AppFooter`; authenticated pages get `AppSidebar` + `AppTopBar`
- Chat thread history moved from inline `<aside>`/`<Sheet>` into the global sidebar via Zustand store
- Nav links removed from `AppHeader` since signed-in users now navigate via sidebar
- Homepage redirects signed-in users to `/chat` (server-side + client-side `AuthRedirect`)

## Current State

- **Build passes** (`pnpm build` — zero type errors)
- **Lint passes** (`pnpm lint` — zero ESLint errors, 1 Biome warning on vendor code)
- **`check:docs` passes** after doc header fixes
- **2 unit tests failing** in `tests/unit/app/app-layout-auth-guard.test.tsx` — the authenticated render cases now pull in the full sidebar/topbar tree which requires `QueryClientProvider` and more mocks. See Next Actions.
- **3 unstaged files** with partial fixes: `AuthRedirect.tsx` (doc header fix), `page.tsx` (doc header fix), `app-layout-auth-guard.test.tsx` (incomplete mock additions — `usePathname` added, `QueryClientProvider` wrapper started but not finished)
- **Visual regression: flashy redirect on logo click** — when signed-in user clicks the Cogni logo (links to `/`), they briefly see the public layout (header+footer, no sidebar) before `AuthRedirect` fires and navigates to `/chat`. This is jarring. The redirect approach may need rethinking — see Risks.

## Decisions Made

- **Zustand store** bridges chat thread state from `chat/page.tsx` to `AppSidebar.tsx` (sidebar is a sibling, not ancestor — React context won't work)
- **`chat-viewport` CSS restored** — `calc(100dvh - var(--app-header-h))` provides absolute height constraint for the chat area, same proven approach as pre-sidebar
- **`overflow-auto` on app content wrapper** — non-chat pages (gov, activity) need vertical scrolling; chat layout overrides with `overflow-hidden`
- **Root `<main id="main">` → `<div id="main">`** — `SidebarInset` renders its own `<main>`, preventing nested `<main>` HTML violation
- **Sidebar tokens added to ESLint allowlist** — `sidebar-*` is a legitimate design system color category (`scripts/eslint/plugins/ui-governance.cjs`)
- **All vendor imports go through `@/components` barrel** — 4 files fixed, `SidebarMenuAction` added to barrel exports

## Risks / Gotchas

- **Logo redirect flash** is the most user-visible regression. The two-layout architecture (public vs app) means navigating between them always briefly shows the wrong shell. Next.js middleware (`middleware.ts`) may be the cleanest fix — server-side redirect avoids client-side flash entirely.
- **`SidebarProvider` sets a cookie** (`sidebar_state`) for persistence — Biome warns about `document.cookie` in vendor code. Leave vendor code unmodified per project policy.
- **`exactOptionalPropertyTypes: true`** in tsconfig means optional props must explicitly include `| undefined` — previously bit with `ThreadSummary.title`.
- **Chat Zustand store** registers/unregisters on mount/unmount — if chat page unmounts before sidebar reads, thread group may flash empty. Test navigation away from `/chat`.
