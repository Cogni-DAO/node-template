## Risks / Gotchas

- `COGNI_REPO_PATH=.` in `.env.test` leaks into Docker containers via `docker compose --env-file .env.test`, overriding the compose default `/repo/current`. This is the root cause of the `check:full` readyz 500 and may have been broken on staging pre-refactor too.
- Any test or production code using `process.cwd()` to find repo-root files will break — must use `serverEnv().COGNI_REPO_ROOT` instead. Three files were fixed; grep for `process.cwd()` in `apps/operator/src/` to check for more.
- The scheduler-worker Dockerfile's fragile multi-COPY approach (lines 82-110) depends on root `package.json` listing all workspace deps. Adding a new `@cogni/*` package requires updating root `package.json` too, until `pnpm deploy` replaces this pattern.
- Stack tests using `grep -rn ... src/` or `execSync` with `cwd: process.cwd()` need `join(process.cwd(), "apps/operator")` since tests run from repo root but `src/` moved.
- `pnpm install` is needed after the root `package.json` workspace dep changes — lockfile was updated but verify with `--frozen-lockfile` in CI.
